# 🏓 Table Tennis Ball Tracking System  
## Team Nirmaan51

A computer vision–based table tennis analytics system that detects and tracks the ball, maps its position to real-world table coordinates using homography, automatically detects bounce events using velocity analysis, maps impacts to a 3×4 haptic motor grid, and exports structured match data.

---

## 📌 Overview

This project processes a recorded table tennis match video and:

- Detects the ball using HSV color segmentation (yellow/orange or white)
- Removes table color interference (auto-calibrated)
- Tracks ball position frame-by-frame
- Converts pixel coordinates to real-world table coordinates
- **Automatically detects bounces** using vertical velocity inversion
- Maps bounce impacts to a 3×4 haptic motor grid
- Generates annotated output video
- Logs structured event data into CSV

---

## 🎯 Features

- 🎥 Real-time ball detection using OpenCV
- 📐 Homography-based coordinate transformation
- 🏓 **Automated bounce detection** (velocity inversion + peak tracking)
- 🎨 **Dynamic color calibration** (table color sampled, ball color selectable)
- 🟦 3×4 haptic grid mapping for motor feedback
- 📊 Automatic CSV event logging
- 💾 Processed output video generation

# 🚀 How to Use

## 🔹 Option 1: View Demo Output (No Setup Required)

If you only want to see the system output:

1. Open:
```
processed_output.mp4
```
2. View the annotated ball tracking, bounce grid, and hit detection.

No installation required.

---

## 🔹 Option 2: Run the System Yourself

### 1️⃣ Clone the Repository

```
git clone https://github.com/Jasz-rgb/DesignForBharat-OpenCV.git
cd DesignForBharat-OpenCV
```

---

### 2️⃣ (Recommended) Create a Virtual Environment

```
python -m venv venv
```

Activate it:

**Windows**
```
venv\Scripts\activate
```

**Mac/Linux**
```
source venv/bin/activate
```

---

### 3️⃣ Install Dependencies

```
pip install -r requirements.txt
```

Your `requirements.txt` contains:

```
opencv-python>=4.8.0
numpy>=1.24
```

---

### 4️⃣ Calibrate Table & Ball Color

Run calibration (pass your video file as argument):

```
python caliberate_table.py input.mp4
```

Click 5 points on the frame:

1. Bottom-Left corner  
2. Bottom-Right corner  
3. Top-Right corner  
4. Top-Left corner  
5. **Click inside the table** (for color sampling)  

Then select ball color in terminal:
- `1` — Yellow / Orange (default)
- `2` — White

This generates:

```
table_corners.npy
calibration_data.json
```

---

### 5️⃣ Run Ball Tracking

```
python main.py input.mp4
```

The video file argument is optional — if omitted, it uses the video from `calibration_data.json`.

Press **ESC** to exit the video window.

### 📄 CSV File

`ball_events.csv`

Contains structured bounce data with the following columns:

- `frame` — frame number
- `event` — event type (BOUNCE)
- `grid_col` — motor grid column (0–2)
- `grid_row` — motor grid row (0–3)
- `table_x` — real-world X coordinate (metres)
- `table_y` — real-world Y coordinate (metres)

This file is automatically generated after running the main script.

## 📐 Calibration Details

The system uses homography mapping to convert camera pixel coordinates into real-world table dimensions.

Official table dimensions used:

- Width: **1.525 meters**
- Length: **2.74 meters**

Calibration data is stored in:

```
calibration_data.json    # corners, table HSV, ball color, video path
table_corners.npy        # corner points (legacy format)
```

## 🧠 How It Works

1. Convert frame from BGR to HSV color space  
2. Apply calibrated color threshold to detect ball (yellow/orange or white)  
3. Remove table color interference (auto-sampled during calibration)  
4. Apply morphological filtering  
5. Detect contours and identify valid ball contour  
6. Transform pixel coordinates to real-world table coordinates via homography  
7. Feed Y-coordinate into bounce detector (velocity inversion with peak tracking)  
8. On bounce: validate on-table, map to 3×4 grid, trigger haptic board via serial  
9. Overlay visualization (impact marker, grid position, phase)  
10. Save processed video and event logs
